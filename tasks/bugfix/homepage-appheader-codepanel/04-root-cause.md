# 04. 根本原因分析

## 问题本质

三个问题的根本原因各不相同，但都属于**前端组件生命周期和状态管理**层面的缺陷：

1. **列头遗漏**: 某次重构中 ProjectCard 添加了 Docs 列，但 HomePage 的 `<thead>` 未同步更新
2. **生命周期误用**: AppHeader 错误地将 Store 级别的资源 (SSE 连接) 绑定到组件级生命周期
3. **并发控制缺失**: CodePanel 缺少异步请求的并发控制机制

---

## 问题 1: HomePage 表格列头遗漏

### 调用链分析

```
HomePage.vue <table>
    │
    ├── <thead>
    │     └── <tr>
    │           ├── <th> 项目名称      ✅
    │           ├── <th> 仓库类型      ✅
    │           ├── <th> 最近更新      ✅ (但应该是 Docs)
    │           ├── <th> 状态          ✅ (但应该是最近更新)
    │           └── <th> (空)          ✅ (但应该是状态)
    │           ❌ 缺少: <th> Docs
    │
    └── <tbody>
          └── <ProjectCard> (v-for)
                └── <tr>
                      ├── <td> 项目名 + 路径     ← 列 1
                      ├── <td> Git / GitHub      ← 列 2
                      ├── <td> docCount (33)     ← 列 3 (没有对应列头!)
                      ├── <td> 最近更新           ← 列 4
                      ├── <td> 分析状态           ← 列 5
                      └── <td> 操作按钮           ← 列 6
```

**根因**: ProjectCard 在某次迭代中添加了 Docs 列 (`docCount`)，但 HomePage 的 `<thead>` 没有同步添加对应的 `<th>`。

**为什么没发现**:
1. HTML 表格不强制列数匹配 — 浏览器会自动渲染多余列
2. 没有视觉测试 (screenshot comparison) 验证表格布局
3. 开发时可能只关注了单行数据，没注意到列头错位

## 问题 2: AppHeader SSE 连接泄漏

### 生命周期时序分析

```
Vue 3 组件生命周期 (导航时):
┌─────────────────────────────────────────────────────────────┐
│  Step 1: 新组件 mounted                                     │
│  ┌──────────────────────────────────────────────────┐       │
│  │  ProjectLayout.AppHeader.onMounted()              │       │
│  │      ↓                                            │       │
│  │  analysisStore.connectSSE() → 新 EventSource 创建  │       │
│  └──────────────────────────────────────────────────┘       │
│                                                             │
│  Step 2: 旧组件 unmounted                                   │
│  ┌──────────────────────────────────────────────────┐       │
│  │  HomePage.AppHeader.onUnmounted()                 │       │
│  │      ↓                                            │       │
│  │  analysisStore.disconnectSSE() → 关闭 EventSource  │ ← BUG │
│  └──────────────────────────────────────────────────┘       │
│                                                             │
│  结果: Step 1 创建的连接被 Step 2 关闭!                       │
└─────────────────────────────────────────────────────────────┘
```

### 资源归属分析

```
SSE 连接应该归属于:
  ✅ Store (全局单例) → 跟随应用生命周期
  ❌ Component (局部实例) → 跟随组件生命周期

AppHeader 错误地将 Store 资源绑定到组件生命周期:
  onMounted → connectSSE()   ← 可以 (首次建立)
  onUnmounted → disconnectSSE() ← 错误! (组件销毁不应影响全局 Store)
```

**根因**: AppHeader 的 `onUnmounted` 调用了 `analysisStore.disconnectSSE()`，但 SSE 连接是 Store 级别的全局资源，应该在整个应用退出时才断开，而不是在组件销毁时。

**为什么没发现**:
1. 单页应用中，导航操作频繁但用户很少注意到 SSE 重连的瞬间中断
2. `connectSSE()` 内部有幂等处理 (先 close 旧连接再创建新连接)，所以功能上不会报错
3. 只是增加了不必要的重连开销和状态闪烁

## 问题 3: CodePanel 竞态条件

### 竞态时序分析

```
用户快速点击 file-a 和 file-b:

时间    事件                          fileRequestCounter    currentFile
─────────────────────────────────────────────────────────────────────
T0      点击 file-a                   1                     file-a
T0.1    fetch(file-a) 开始            1                     file-a
T1      点击 file-b                   2                     file-b
T1.1    fetch(file-b) 开始            2                     file-b
T2      fetch(file-b) 完成            2                     file-b  ✅ 正确
T3      fetch(file-a) 完成            2                     file-a  ❌ 覆盖!
─────────────────────────────────────────────────────────────────────

修复后 (使用 fileRequestCounter):

时间    事件                          fileRequestCounter    reqId 检查    结果
─────────────────────────────────────────────────────────────────────────────
T0      点击 file-a                   1                     -            -
T0.1    fetch(file-a) 开始            1                     reqId=1      -
T1      点击 file-b                   2                     -            -
T1.1    fetch(file-b) 开始            2                     reqId=2      -
T2      fetch(file-b) 完成            2                     2===2 ✅      显示 file-b
T3      fetch(file-a) 完成            2                     1!==2 ❌      丢弃
─────────────────────────────────────────────────────────────────────────────
```

**根因**: `selectFile()` 和 `loadFile()` 中的 `await fetch()` 没有检查请求是否仍然有效。当用户快速切换文件时，旧请求的响应会覆盖新请求的结果。

**为什么没发现**:
1. 正常操作速度下竞态几乎不会发生
2. 只有在快速连续点击时才能复现
3. 没有自动化测试模拟快速点击场景

```
┌─────────────────────────────────────────────────────────────────────┐
│                        三个问题的架构根因                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                 │
│  问题 1: 组件间契约不一致                                         │
│  ┌──────────────┐         ┌──────────────┐                       │
│  │  HomePage    │ ──???──→│ ProjectCard  │                       │

│  └──────────────┘         └──────────────┘                       │

│                                                                 │
│  问题 2: 资源归属错误                                             │
│  ┌──────────────┐                                                │
│  │  AppHeader   │──owns──→ SSE 连接 (应该是 Store owns)          │
│  │  (组件级)     │         (全局资源)                              │
│  └──────────────┘                                                │
│  根因: 组件错误地管理了不属于它的资源                               │
│                                                                 │
│  问题 3: 异步操作无并发控制                                       │
│  ┌──────────────┐                                                │
│  │  CodePanel   │──spawn──→ fetch(file-a) ──┐                    │
│  │              │──spawn──→ fetch(file-b) ──┤← 竞态!             │
│  └──────────────┘                           │                    │
│  根因: 没有请求取消/版本检查机制                                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────────┘
```

## 修复策略

| 问题 | 修复方式 | 涉及文件 | 代码量 |
|------|----------|----------|--------|
| 列头遗漏 | 添加缺失的 Docs `<th>` | HomePage.vue | +1 行 |
| SSE 断开 | 移除 onUnmounted 中的 disconnectSSE | AppHeader.vue | -3 行 |
| 竞态条件 | 添加 fileRequestCounter | CodePanel.vue | +10 行 |
