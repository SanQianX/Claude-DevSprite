## 改进建议

### 1. E2E 测试覆盖

当前 5 个 UI 测试覆盖了主要功能，但缺少：
- 审查扫描的端到端流程 (点击扫描 → 等待结果 → 验证列表更新)
- 修复流程的端到端验证 (点击批准修复 → 验证文件被修改)
- 分析流程的进度验证 (点击分析 → SSE 进度 → 完成状态)

### 2. API 客户端完整性检查

建议在 CI 中添加检查：
```bash
# 检查后端端点是否都有对应的前端 API 方法
grep -r "app\.\(post\|get\|put\|delete\)('/api/" src/worker/routes/ | \
  while read line; do
    endpoint=$(echo "$line" | grep -o "'/api/[^']*'")
    # 检查 web/src/api/ 中是否有对应方法
  done
```

### 3. 组件调用链审计

建议定期审计：
- 所有 `dashboardApi.*` 方法是否都被 store 或组件调用
- 所有 store 方法是否都被组件调用
- 所有按钮事件是否都有对应的处理函数

### 4. 命名一致性

当前存在命名混乱：
- `approveReview` 只批准不修复 → 应改为 `markApproved`
- `fixReview` 实际修复 → 保持不变
- 建议统一命名: `approve` (只设状态) vs `fix` (实际修复)
