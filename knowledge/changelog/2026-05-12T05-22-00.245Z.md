### 4. web/src/api/dashboard.ts

**变更类型**: 修改

**变更详情**:

在仪表盘 API 客户端中新增了两个接口函数：

1.  **`triggerScan`**: 调用 `/projects/{projectName}/reviews/scan` 接口，触发对指定项目的 AI 审查扫描，并返回发现的问题数量 (`findingsCount`)。
2.  **`fixReview`**: 调用 `/reviews/{reviewId}/fix` 接口，触发对指定审查项的修复操作，并返回修复说明 (`explanation`)。

**影响范围**: 前端 API 层。为前端界面新增的交互按钮提供了必要的网络请求能力，是连接 UI 与后端逻辑的桥梁。

**相关文件**: [dashboard.ts](/project/Claude-DevSprite/source?path=web/src/api/dashboard.ts)

### 5. web/src/stores/dashboard.ts

**变更类型**: 修改

**变更详情**:

对仪表盘的状态管理逻辑进行了优化和功能增强：

1.  **移除冗余注释**: 移除了 `approveReview` 函数中一条关于状态更新策略的旧注释。
2.  **新增 `fixReview` 函数**:
    - 调用新的 `dashboardApi.fixReview` 接口执行修复。
    - 在前端状态（`reviews` 数组）中查找对应的审查记录。
    - 更新该记录的 `status` 为 `"fixed"`，并设置 `resolved_at` 为当前时间。
    - 更新统计数据 (`reviewStats`)，增加已修复 (`fixed`) 的计数，并减少待审批 (`pending`) 的计数。

**影响范围**: 前端状态管理核心逻辑。实现了审查修复操作的完整闭环，确保了用户操作后界面状态的正确同步。

**相关文件**: [dashboard.ts](/project/Claude-DevSprite/source?path=web/src/stores/dashboard.ts)

### 6. web/src/views/DashboardView.vue

**变更类型**: 修改

**变更详情**:

对仪表盘视图进行了功能增强和交互优化：

1.  **新增“开始分析”按钮**:
    - 在统计区域（`stat-section`）添加了一个按钮。
    - 按钮文案会根据 `analyzing` 状态在“分析中...”和“开始分析”之间切换。
    - 按钮在分析进行时禁用 (`disabled`)。
    - 点击触发 `startAnalysis` 方法。
2.  **新增“开始扫描”按钮**:
    - 在“AI 审查队列”区域（`review-section`）的标题栏添加了一个按钮。
    - 按钮文案会根据 `scanning` 状态在“扫描中...”和“开始扫描”之间切换。
    - 按钮在扫描进行时禁用 (`disabled`)。
    - 点击触发 `startScan` 方法。

**影响范围**: 用户界面。直接为用户提供了手动触发“项目分析”和“AI审查扫描”操作的入口，提升了 Dashboard 的主动性和可操作性。

**相关文件**: [DashboardView.vue](/project/Claude-DevSprite/source?path=web/src/views/DashboardView.vue)

## 修复的问题列表

本次提交集中修复和增强了以下功能：

1.  **AI 审查修复流程未调用实际端点**: 之前的“批准修复”操作可能未正确调用后端的修复接口。本次修复在前端 API 层 (`fixReview`) 和状态管理 (`fixReview`) 中实现了对 `/reviews/{reviewId}/fix` 端点的完整调用。
2.  **缺少手动触发 AI 审查扫描的入口**: 仪表盘界面没有提供手动触发扫描的按钮。本次修复在 API 层 (`triggerScan`) 和视图层 (`startScan`) 添加了完整的触发机制。
3.  **缺少手动触发项目分析的入口**: 仪表盘界面没有提供手动触发项目分析的按钮。本次修复在视图层 (`startAnalysis`) 添加了触发机制，并可能关联了后端分析接口。
4.  **UI 测试脚本可维护性**: 优化了 DevChat 的 UI 测试脚本，提取了重复配置，提升了可维护性。

## 影响评估

- **功能完整性**: 补全了 Dashboard 的核心交互功能，使“AI审查扫描”和“项目分析”流程可以通过用户界面手动触发，形成了更完整的操作闭环。
- **用户体验**: 增加了操作的主动性和即时性，用户无需等待自动触发，可以按需启动扫描或分析任务。按钮的状态反馈（禁用/文案变化）提升了操作的确定性。
- **代码质量**: API 层、状态管理层和视图层的改动逻辑清晰，职责分离。新增函数遵循了现有代码模式。测试脚本的重构提升了代码质量。
- **系统稳定性**: 本次修复主要增强了前端交互功能，对后端服务的依赖是调用现有接口，因此不会直接影响后端稳定性。修复了“批准修复”可能失效的潜在问题。
