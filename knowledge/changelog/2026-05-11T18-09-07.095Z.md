## 修复的问题列表

## 根本原因

根据排查文件列表 (`reviews.ts`, `DashboardView.vue`) 推断，根本原因可能涉及以下方面：

1.  **后端 API 逻辑缺陷**：`src/worker/routes/reviews.ts` 中处理审查扫描的路由或服务逻辑可能存在错误，导致扫描任务状态更新异常、结果数据格式不正确或查询失败。
2.  **前后端交互问题**：前端 `web/src/views/DashboardView.vue` 调用后端 API 触发扫描或获取结果时，参数传递、响应处理或状态管理可能存在不一致，导致 UI 无法正确反映扫描状态或显示结果。
3.  **异步流程处理不当**：扫描是一个异步过程，可能在任务提交、进度轮询、结果获取的环节中存在竞态条件或未正确处理的异常。

## 修复方案

修复方案的具体实现细节需要查看相关源文件的 diff。根据任务描述推测，修复可能涉及：

1.  **后端修复**：修正 `reviews.ts` 中扫描任务的创建、状态更新、结果查询等核心逻辑，确保数据流的正确性和一致性。
2.  **前端修复**：调整 `DashboardView.vue` 中与扫描交互相关的 UI 组件逻辑，如发起扫描请求、监听进度、接收并展示最终结果的代码，保证交互流程的顺畅。
3.  **端到端流程验证**：确保从用户在界面上点击触发扫描，到看到进度条，再到最终查看扫描详情，整个链条都能正常工作。
