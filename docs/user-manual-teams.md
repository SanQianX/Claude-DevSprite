## 系统概述

多 Agent Team 协作系统是 Claude-DevSprite 的核心功能之一，允许用户通过自然语言与 AI 团队交互，完成软件开发任务。系统采用三人团队模式：

- **Team Lead (领导)**: 分析需求、分配任务、协调团队
- **Developer (开发)**: 编写代码、实现功能
- **Tester (测试)**: 编写测试、验证功能

三个角色通过 Claude CLI 子进程并行执行，通过文件协议进行通信，通过 SSE 实现实时状态推送。

---

## 团队角色

### Team Lead (领导)

**职责:**
- 接收用户需求
- 分析任务复杂度
- 将任务拆分为子任务
- 分配给 Developer 和 Tester
- 汇总结果并回复用户

**特点:**
- 使用 Claude Sonnet 模型
- 只读权限，不直接修改代码
- 协调团队工作流程

### Developer (开发)

**职责:**
- 接收 Lead 分配的任务
- 阅读和理解代码
- 编写和修改代码
- 完成后向 Lead 汇报

**特点:**
- 使用 Claude Sonnet 模型
- 完整读写权限
- 可执行 Bash 命令

### Tester (测试)

**职责:**
- 接收 Lead 分配的测试任务
- 编写测试用例
- 运行测试验证功能
- 报告测试结果

**特点:**
- 使用 Claude Sonnet 模型
- 只读 + 测试权限
- 可执行测试命令

## 快速开始

### 1. 启动服务

```bash
# 启动 Worker 服务
node dev-scripts/daemon.js restart
```

### 2. 访问界面

打开浏览器访问: `http://localhost:3000`

点击左侧导航栏的 **DevChat** 进入团队聊天界面。

### 3. 发送任务

在聊天输入框中输入你的开发任务，例如:

```
请帮我实现一个用户登录功能，包括：
1. 登录表单组件
2. 登录 API 接口
3. JWT token 管理
```

### 4. 观察执行

- 左侧面板显示团队状态（绿色=空闲，黄色=忙碌，红色=错误）
- 右侧聊天区实时显示各团队的消息
- 任务完成后，Lead 会汇总结果

### 消息类型

| 类型 | 样式 | 说明 |
|------|------|------|
| 用户消息 | 蓝色气泡 | 你发送的消息 |
| Agent 消息 | 绿色气泡 | AI 团队的回复 |
| 工具消息 | 灰色气泡 | 工具执行记录 |
| 系统消息 | 居中灰色 | 系统通知 |
| 错误消息 | 红色气泡 | 错误信息 |

### 状态指示器

| 状态 | 颜色 | 说明 |
|------|------|------|
| 空闲 (idle) | 绿色 | 团队成员等待任务 |
| 忙碌 (busy) | 黄色 | 正在执行任务 |
| 错误 (error) | 红色 | 执行出错 |
| 离线 (offline) | 灰色 | 未连接 |

## 使用流程

### 基本开发流程

1. **提出需求**: 在聊天框输入你的开发任务
2. **Lead 分析**: Team Lead 分析需求并制定计划
3. **任务分配**: Lead 将任务分配给 Dev 和 Test
4. **并行执行**: Dev 编写代码，Test 编写测试
5. **结果汇总**: Lead 汇总各方结果
6. **完成回复**: Lead 向用户报告完成情况

### 示例对话

**用户:**
```
请帮我实现一个简单的待办事项列表功能
```

**Lead:**
```
收到！我来分析一下任务：

1. 前端：TodoList.vue 组件
2. 后端：todos.ts API 路由
3. 测试：单元测试 + 集成测试

正在分配任务给 Dev 和 Test...
```

**Dev:**
```
开始实现 TodoList.vue 组件...
- 创建组件结构
- 实现添加/删除/完成功能
- 添加本地存储
```

**Test:**
```
开始编写测试用例...
- 测试添加待办
- 测试删除待办
- 测试状态切换
```

**Lead:**
```
任务完成！

已完成：
✅ TodoList.vue 组件
✅ todos.ts API 路由
✅ 12 个测试用例全部通过

可以开始使用了。
```

## 配置说明

### 团队配置文件

团队配置存储在项目的 `.claude/teams/` 目录下:

```
.claude/
└── teams/
    ├── lead.yml
    ├── dev.yml
    └── test.yml
```

### 配置示例 (lead.yml)

```yaml
name: lead
model: claude-sonnet-4-20250514
role: Team Lead
systemPrompt: |
  你是团队的领导，负责：
  1. 分析用户需求
  2. 制定开发计划
  3. 分配任务给团队成员
  4. 汇总结果并回复用户

  你有以下工具可用：
  - Read: 读取文件
  - Edit: 编辑文件
  - Bash: 执行命令

tools:
  - Read
  - Edit
  - Bash
```

### 修改配置

通过 API 修改团队配置:

```bash
curl -X PUT http://localhost:3000/api/teams/dev \
  -H "Content-Type: application/json" \
  -d '{
    "model": "claude-opus-4-20250514",
    "systemPrompt": "自定义提示词..."
  }'
```

## 常见问题

### Q: 为什么团队成员一直处于"忙碌"状态？

**A:** 可能原因：
1. 任务复杂度高，需要较长时间
2. Claude CLI 进程卡住
3. 网络连接问题

解决方法：
- 点击"中止所有"按钮停止当前任务
- 检查 Worker 日志: `tail -f dev-scripts/worker-stdout.log`

### Q: 如何查看团队执行的日志？

**A:** 通过 API 查看:
```bash
curl "http://localhost:3000/api/logs?lines=100&level=INFO"
```

### Q: 团队成员使用什么 AI 模型？

**A:** 默认使用 Claude Sonnet。可以通过配置文件或 API 修改每个团队的模型。

### Q: 可以同时运行多个任务吗？

**A:** 目前系统设计为串行执行。发送新任务时，如果上一个任务还在执行，需要等待完成或中止。

### Q: 如何中止正在执行的任务？

**A:** 两种方式：
1. 点击界面上的"中止所有"按钮
2. 通过 API: `POST /api/teams/abort-all`

### Q: 文件通信协议是什么？

**A:** 团队间通过文件进行通信:
- **Inbox**: 接收任务的目录 (`.claude/teams/:name/inbox/`)
- **Outbox**: 发送结果的目录 (`.claude/teams/:name/outbox/`)
- 文件格式: Markdown + YAML frontmatter

### Q: SSE 连接断开怎么办？

**A:** 刷新页面重新建立连接。系统会自动重连。

## 高级功能

### 自定义系统提示词

可以通过修改配置文件自定义团队成员的行为:

```yaml
# dev.yml
systemPrompt: |
  你是一个专注于 React 开发的工程师。
  请始终使用函数组件和 Hooks。
  使用 TypeScript 编写代码。
  遵循项目的 ESLint 规则。
```

### 集成 Git

团队可以与 Git 集成，自动提交代码:

```
用户: 请帮我实现登录功能，并提交到 feature/login 分支
```

### 查看执行历史

通过 API 查看历史执行记录:

```bash
# 获取所有团队状态
curl http://localhost:3000/api/teams/status/all

# 获取开发日志
curl http://localhost:3000/api/projects/:name/analysis-log
```
