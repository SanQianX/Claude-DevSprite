## 1. 测试环境

| 角色 | 地址 | 说明 |
|------|------|------|
| 本地机器 | `http://127.0.0.1:38888` | 办公室电脑，运行完整系统 |
| 云服务器 | `http://124.220.17.38:38888` | 腾讯云，Ubuntu 24.04，中继角色 |
| 浏览器 | 任意设备 | 通过服务器地址访问 |

### 1.1 设备识别

**问题**: 本地和远程运行相同程序，无法区分哪个是哪个。

**解决方案**: 在设置页面添加"设备角色"配置项：
- 本地机器设为 `local`（执行者）
- 服务器设为 `relay`（中继者）
- 页面 Header 显示角色标签（如 "🖥 本地" 或 "☁️ 中继"）

## 2. 测试场景

### 场景 1: 页面一致性（远程 = 本地的镜像）

**目标**: 远程浏览器看到的页面与本地完全一致

**测试步骤**:
1. 本地打开 `http://127.0.0.1:38888`，截图所有页面
2. 远程打开 `http://124.220.17.38:38888`，截图相同页面
3. 逐页对比差异

**通过标准**:
- 项目列表数据一致（名称、类型、文档数、更新时间）
- 任务列表一致（数量、标题、状态）
- 审查列表一致（数量、标题、严重级别）
- 页面布局、导航结构一致

**当前状态**: ❌ 不一致

| 差异项 | 本地 | 远程 | 原因 |
|--------|------|------|------|
| 仓库类型 | Git | Local | sync 数据缺少 repositoryType 字段 |
| 文档数 | 65 | 0 | sync 数据缺少 documentCount 字段 |
| 最近更新 | Today | Never | sync 数据缺少 lastUpdated 字段 |

**修复方案**: syncClient 发送 full state 时补充缺失字段

---

### 场景 2: 实时同步（本地操作 → 远程立即可见）

**目标**: 本地任何状态变更，远程浏览器自动更新

**测试步骤**:
1. 打开两个浏览器窗口：本地 + 远程
2. 在本地触发操作（如创建任务、触发扫描）
3. 观察远程是否自动更新（无需刷新）

**测试用例**:

| # | 操作 | 本地预期 | 远程预期 | 验证方式 |
|---|------|----------|----------|----------|
| 2.1 | 创建任务 | 任务列表增加 | 自动出现新任务 | 截图对比 |
| 2.2 | 完成任务 | 状态变为 done | 状态同步更新 | 截图对比 |
| 2.3 | 触发扫描 | 扫描状态变为 running | 状态同步更新 | 截图对比 |
| 2.4 | 发送聊天 | 消息出现在聊天中 | 聊天消息同步 | 截图对比 |
| 2.5 | 删除审查 | 审查列表减少 | 自动减少 | 截图对比 |

**通过标准**: 远程在 3 秒内看到变更，无需手动刷新

**当前状态**: ❌ 不工作
- 原因: `handleSyncFull` / `handleSyncState` 只有 console.log，未应用到 store

---

### 场景 3: 远程命令中继（远程操作 → 本地执行）

**目标**: 在远程浏览器点击按钮，本地机器执行对应操作

**测试步骤**:
1. 远程浏览器打开系统
2. 点击"开始扫描"按钮
3. 检查本地是否开始扫描（查看本地 UI 或日志）

**测试用例**:

| # | 远程操作 | 本地预期行为 | 验证方式 |
|---|----------|-------------|----------|
| 3.1 | 点击"开始扫描" | 本地启动扫描进程 | 截图本地扫描状态 |
| 3.2 | 发送聊天消息 | 本地 Claude CLI 执行 | 截图本地聊天输出 |
| 3.3 | 点击"开始修复" | 本地启动修复 | 截图本地修复状态 |
| 3.4 | 修改配置 | 本地配置更新 | 截图本地设置页 |

**通过标准**: 本地在 5 秒内开始执行对应操作

**当前状态**: ❌ 不工作
- 原因: AgentStatusPanel 未挂载，命令中继 UI 入口缺失
- relay 命令 (scan.trigger, fix.trigger) 服务端已实现，但前端无触发入口

---

### 场景 4: Agent 离线/重连

**目标**: 本地关机后远程看到离线提示，重启后自动恢复

**测试步骤**:
1. 关闭本地 Agent 进程
2. 远程浏览器应显示"Agent 离线"提示
3. 重启本地 Agent
4. 远程浏览器应自动恢复连接并同步数据

**通过标准**:
- 离线时: 远程显示离线状态，历史数据仍可查看
- 重连时: 自动全量同步，无需手动操作

**当前状态**: ⚠️ 部分工作
- AgentStatusPanel 组件已实现但未挂载
- syncServer 的 agent.online/offline 广播已实现

---

### 场景 5: 设备角色识别

**目标**: 用户能明确知道当前看到的是本地还是远程

**测试步骤**:
1. 打开远程浏览器
2. 检查页面是否显示"中继模式"或类似标识
3. 打开本地浏览器
4. 检查页面是否显示"本地模式"

**通过标准**:
- 页面 Header 或明显位置显示设备角色
- 设置页显示当前模式（本地/中继）
- 远程浏览器标题显示 "Relay - Claude-DevSprite"

**当前状态**: ❌ 未实现
