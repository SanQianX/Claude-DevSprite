Project Workspace UI 设计稿

Claude-DevSprite 项目页面重新设计 — 各状态 UI 示意图

📊 Dashboard 模式
📊
Dashboard 默认状态
项目信息栏 + 项目计划（任务列表：进行中/已完成/待开发）+ 进度统计 + AI 审查队列（待审批列表，含严重性标记和操作按钮）
Dashboard 新功能
🔍
审查项详情展开
点击审查项后展开详情：问题描述、AI 修复建议（Diff 视图）、元信息、操作按钮、相关审查项
Dashboard
➕
添加任务对话框
点击"+ 添加任务"弹出的对话框：标题、描述、状态、优先级、预估工期
Dashboard
💻 Workspace 模式
📐
三面板模式 (Doc + Code + Chat)
完整的三栏布局：左侧文件树（知识库/源码 Tab 切换）、文档面板（含 source 链接）、代码面板（语法高亮 + 行号 + 高亮行）、对话面板。 文档中的 [source:path:line] 链接点击后自动定位到代码面板对应行。
Workspace 核心
📄
双面板 (Doc + Chat)
关闭 Code 面板后的布局。点击文档中的 source 链接会弹出代码预览浮层，并提示可自动打开 Code 面板。
Workspace
📁
双面板 (Code + Chat)
关闭 Doc 面板后的布局。左侧自动切换到源码文件树，代码面板底部显示关联文档链接。
Workspace
💬
单面板 (仅 Chat)
关闭 Doc 和 Code 面板后的全屏对话模式。支持附带文档/代码上下文的附件栏。
Workspace
🧠 开发记忆 (v1.1.0 新增)
🧠
带开发记忆的三面板
Chat 面板集成开发记忆功能：自动加载相关知识文档、历史会话摘要、关键决策、审查队列项。 记忆横幅显示所有自动加载的上下文，可展开查看详情。每次开发都有延续性，不再是孤立的会话。
Workspace v1.1.0
