## 1. 概述

### 1.1 目标

将项目页面 `/project/:name` 从单一的文档浏览页面，改造为**双模式项目工作台**：

- **Dashboard 模式**：项目总览、任务管理、AI 审查队列
- **Workspace 模式**：文档 + 代码 + 对话的可折叠多面板工作台

同时重新设计**首页 `/`**，移除左侧边栏，新增 Tokens 消费统计，优化整体布局。

### 1.2 核心设计理念

| 原则 | 说明 |
|------|------|
| **面板独立** | 每个面板可独立开关，最少保留 Chat 面板 |
| **布局自适应** | 根据打开的面板数量自动调整布局 |
| **Doc↔Code 关联** | 文档中的代码引用可直接跳转到代码位置 |
| **AI 审查驱动** | AI 后台扫描 → 人工审批 → AI 修复的闭环 |
| **双模式切换** | Dashboard 和 Workspace 通过 Tab 切换 |

## 2. 整体架构

### 2.1 页面结构


## 3. Dashboard 模式

### 3.1 功能描述

Dashboard 是项目进入后的默认页面，提供项目全局视图：

1. **项目信息栏**：项目名称、路径、统计概览
2. **项目计划**：任务的增删改查，状态流转
3. **AI 审查队列**：AI 发现的问题列表，人工审批

### 3.2 项目计划 — 状态流转

```
Backlog → In Progress → Done
   ↑          ↓           ↓
   └──────────┘ (回退)    Archive
```

### 3.3 AI 审查队列 — 交互流程

```
AI 后台扫描 (定时/commit后/手动触发)
        │
        ▼
  发现问题 → 写入队列 (status: pending)
        │
        ├─→ ✅ 批准修复 → AI 自动修复 → 生成变更 → Chat 通知
        ├─→ ❌ 忽略    → 标记 ignored → 从列表隐藏
        └─→ 💬 讨论    → 打开 Chat → 与 AI 讨论方案
```

## 4. Workspace 模式

### 4.1 面板系统

Workspace 由三个独立面板组成，每个面板可开关：

| 面板 | 内容 | 默认状态 |
|------|------|---------|
| **Doc Panel** | 知识库文档浏览 | 关闭 |
| **Code Panel** | 项目源码浏览 | 关闭 |
| **Chat Panel** | 开发对话 | 打开 |

### 4.2 布局自适应规则

```
打开的面板数    布局方式        各面板宽度
─────────────────────────────────────────
    1          单栏全屏        100%
    2          双栏等分        50% | 50%
    3          三栏等分        33% | 33% | 34%

用户可通过拖拽分隔线自定义宽度，双击分隔线恢复均分。
```

### 4.3 Doc ↔ Code 关联机制

#### 文档中的代码引用格式

```markdown
<!-- 格式1: 指定文件 + 行号范围 -->
[source:src/analyzer/pipeline.ts:42-58]

<!-- 格式2: 指定文件 -->
[source:src/analyzer/pipeline.ts]

<!-- 格式3: 带显示文本 -->
Pipeline 类 ([source:src/analyzer/pipeline.ts:42])
```

#### 点击行为

```
用户点击文档中的 [source:...] 链接
        │
        ├─ Code 面板关闭 → 自动打开 Code 面板 → 定位到文件+行号
        ├─ Code 面板打开 → 直接定位到文件+行号
        └─ 目标行高亮 3 秒后渐隐
```

#### 反向关联

```
代码面板底部显示关联文档链接：
  📎 关联文档: 03-modules.md > 分析引擎
  点击 → 切换到 Doc 面板，定位到对应章节
```

## 5A. 开发记忆功能 (Development Memory)

### 5A.1 功能概述

开发记忆功能让 AI 在每次开发会话中自动加载相关上下文，实现**跨会话的开发延续性**。

**核心理念**：每次开发不是从零开始，而是基于已有的知识、历史决策和当前任务继续推进。

### 5A.2 记忆来源 (Context Sources)

| 来源 | 说明 | 加载方式 |
|------|------|---------|
| **当前任务** | Dashboard 中的任务描述、目标、进度 | 关联任务 ID 自动加载 |
| **知识文档** | 与当前任务/代码相关的知识库文档 | 根据任务标签 + 代码路径匹配 |
| **历史会话** | 同一任务下的 previous chat session 摘要 | 按任务 ID 查询最近 N 条会话 |
| **代码变更** | 当前任务涉及的文件变更 (git diff) | 根据任务关联的文件路径 |
| **审查队列** | 与当前任务相关的 AI 审查项 | 按文件路径 + 任务标签匹配 |
| **关键决策** | 历史会话中提取的决策记录 | 从会话摘要中结构化提取 |

### 5A.3 记忆数据模型

### 5A.6 记忆匹配算法

当用户开始新会话时，自动匹配相关上下文：

```typescript
async function buildMemoryContext(
  projectName: string,
  taskId?: string,
  chatInput?: string
): Promise<DevMemoryContext> {
  const context: DevMemoryContext = {
    documents: [],
    previousSessions: [],
    codeChanges: [],
    reviews: [],
    decisions: [],
  }

  // 1. 如果关联了任务，加载任务详情
  if (taskId) {
    context.task = await tasksApi.getTask(taskName, taskId)
  }

  // 2. 根据任务标签 + 用户输入，搜索相关知识文档
  const keywords = extractKeywords(context.task, chatInput)
  context.documents = await searchKnowledgeDocs(projectName, keywords)

  // 3. 查询同一任务下的历史会话摘要
  if (taskId) {
    context.previousSessions = await getSessionSummaries(projectName, taskId)
  }

  // 4. 获取任务关联的代码变更
  if (context.task) {
    context.codeChanges = await getRelatedCodeChanges(projectName, context.task)
  }

  // 5. 匹配审查队列项
  context.reviews = await matchReviews(projectName, keywords, context.documents)

  // 6. 从历史会话中提取决策
  context.decisions = extractDecisions(context.previousSessions)

  return context
}
```

### 5A.8 API 设计 (记忆相关)

```
GET    /api/projects/:name/memory/context?taskId=...
       → 构建并返回 DevMemoryContext
       → 自动匹配知识文档、历史会话、审查项

POST   /api/projects/:name/memory/summary
       → 保存会话摘要 (会话结束时调用)
       → Body: { sessionId, taskId, summary, keyDecisions, filesModified }

GET    /api/projects/:name/memory/summaries?taskId=...
       → 获取指定任务的历史会话摘要列表

GET    /api/projects/:name/memory/decisions?taskId=...
       → 获取指定任务的关键决策列表

POST   /api/projects/:name/memory/search
       → 智能搜索记忆内容
       → Body: { query, taskId?, types?: ('doc'|'session'|'decision'|'review')[] }
```
